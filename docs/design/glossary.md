## 2. Canon 决议 + 校订结果（✅ 已落地 2026-05-31）

### 2.1 「维度」↔「分组」（TagGroup）→ **canon = 分组**
- **决议**：UI 术语统一 **分组**；「维度」仅作概念注解或数据透视轴。
- **校订发现**：逐条核查后，**术语层已符合**——「维度」10 处中 6 处是**数据透视的轴维度**（`pvSplit`/`pvSelectXY`/行列维度，非 TagGroup，**保留正确**），其余是「分组用于…维度分类」这类概念注解（术语已是「分组」）。**无需改动**，强改反会把 pivot 轴误写成「分组」。en 侧同理（Group=术语，dimension=概念/pivot）。

### 2.2 「检索」↔「搜索」（Search）→ **canon = 功能「检索」/ ⌘K「搜索」**
- **决议**：实体多维查询**功能**统一 **检索**；⌘K 命令面板与轻量筛选用 **搜索**。
- **校订改动**：
  - `app-shell` 的 ⌘K 标签原误用 `nav.search`=「实体检索」→ 改用 `common.search`=「搜索」（⌘K 是命令面板，非检索功能）✅
  - `system.enabledHint`「NL 搜索」→「NL 检索」（属检索功能）✅
  - 功能页 nav/title 本已是「实体检索」✓；⌘K 占位「搜索标签、分组、页面…」本已正确 ✓；实体列表 id 快筛保留「搜索」（轻量筛选）✓

### 2.3 其他
- 「登记」vs「注册」：核查未见正文混用，统一「注册」无遗留。
- 无组件硬编码术语（文案全在 messages）。

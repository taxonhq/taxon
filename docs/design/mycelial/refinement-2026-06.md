# 菌丝落地 v2 · 画布纯化与导航收口（2026-06 评估修订）

> **本文修订 `design-notes.md` 的部分落地决策**，不推翻设计语言本身。
> 触发：2026-06 对已落地 console 的逐页评估发现——菌丝识别只活在"首页 hero + 导航"，
> 一进入干活页面（检索/审核/治理）就退回成一个**带边框、低对比的传统深色后台**。
> identity 只是"覆盖"在外层，没有"渗透"进工作流。
>
> 本文把讨论达成的共识锁档，作为 epic #109 后续阶段的执行基准。
> 与 `design-notes.md` 冲突处，**以本文为准**。

---

## 0. 底层原则（一句话宪法）

> **界面是漂浮在无限大地上的物件，不是切进窗口里的盒子。**

本次所有决策都是这条原则的推论。后续任何改动若违背它，即为跑偏。

三个侧面：
- **画布与边界**（删双模式、去页面框、内容靠量度约束）—— 本体
- **漂浮操作层**（spine 漂浮感、hover 全展开）—— 让漂浮层有深度、可认知
- **配置收口**（设置面、logo 回家）—— 把不属于"大地"的东西移走，漂浮层只剩"干活的动词"

---

## 1. 画布唯一形态：删除正常/全屏双模式

- 现状：`app-shell.tsx` 有 `toggleWide`（正常居中 / 宽屏）切换。
- 决策：**删除该切换**。系统只有一种形态——内容漂浮在无限大地上。
- 理由：双模式是传统"max-width 容器"思维的残留，与"大地无限"直接冲突；增加复杂度与编码量，且无正向 UI 收益。

### ⚠️ 关键语义澄清（决定第 4 节怎么做）

**"全屏" ≠ "所有元素拉满 100% 宽"。**

- 大地是无限的（背景大气 / 有机体 / 关系图铺满视口）。
- 但**漂浮在大地上的内容物件仍要有合理"量度（measure）"**——一张表在 3440px 宽屏拉满 = 一行 200 字不可读。
- 所以："容器边框"没了，但**"约束内容宽度"这个职责，从『边框』转交给『物件自身的尺寸纪律』**。

---

## 2. 去页面外框：漂浮玻璃物件取代"玻璃 sheet 大框"

> 修订 `design-notes.md` §5「密集页 = 玻璃 sheet」。当前 sheet 被实现成了**一整块带边框的大盒子**，
> 视觉上等于"在大地上挖个洞把内容塞进去"，是廉价感/"回到传统后台"的最大来源。

- 决策：**删掉页面级外框**。功能性分组改用**漂浮玻璃物件**表达。
- 区分两种东西（这是成败关键）：
  - ❌ **框 = 窗口**：在大地上挖洞塞内容（当前做法）
  - ✅ **卡 = 物件**：一块漂浮在土壤之上的玻璃，有自己的影子与呼吸边距
- 原来"边框约束宽度"的活，交给第 1 节的**量度纪律**。
- ⚠️ 注意："去框" ≠ "去掉一切容器"。删的是**页面级外框**，不是功能分组的玻璃物件。

### 实现期必须同时解决（否则"无框 + 低对比"会更飘）

- **暗色对比度校准**：正文 `--dim` 提向 `--cream`；玻璃透明度降一档。
  （`design-notes.md` §8 已自警 `--dim` 小字过不了 WCAG，本次一并修。）
- **顶部行的层次**：没了边框，页面标题/操作行靠什么"收口"在大地上——
  需要一套留白/对齐规范替代边框职责，避免做散。

---

## 3. 配置收口：右上角统一设置面（Chrome 式）

> 修订 #109「设置三页（LLM / 系统 / Tokens）→ sheet」。不再是三页，而是**收口成一个设置面**。

- 决策：右上角一个**设置入口** → 单页 + 左侧分节锚点（`chrome://settings` 模型）。
- 收纳：**LLM 设置 / API Token / 事件订阅 / 系统设置 / 版本信息 / 默认配色 / API 文档**。
- 原则：**"使用系统"与"配置系统"是两类动作，不该并列在导航里。**
- 收益：spine 从 10 项砍掉 4 个设置类目；页面数量下降；操作更聚焦。
- ⚠️ 设置面**自身也遵守第 2 节"无外框"**，别又做成大盒子。

---

## 4. 导航收敛：spine 10 → 5

- **logo 回首页**：点左上 `Taxon` 回 dashboard（标准 web 约定）→ spine 去掉 dashboard 项。
- **设置收口**（第 3 节）→ 砍掉 LLM/Token/事件订阅/系统 四项。
- 结果：**spine 最终 = 分组 / 实体 / 检索 / 审核 / 治理**（五个核心动词）。
- 配套：`⌘K` 命令面板**升为一等公民**——它是"去任何深层页面/执行任何动作"的主入口，
  与精简导航天生一对（现在它在右下角是弱提示，地位要提上来）。

## 5. 漂浮操作层：让 spine 真正"浮起来" + hover 全展开

### 5.1 漂浮透视感（现状太平）

现状只是"一列发光圆 + 一条细线"，是平的。强化手段（靠光影一致性，不是 z-index）：
- spine 向右下在内容上**投一道柔影**（当前几乎无 cast shadow，所以贴在画面上而非浮起）
- `backdrop-filter` 加重 + 一层很淡的光晕垫在节点下
- 激活/hover 节点**真正"抬起"**（scale + 更深阴影 + 高光点），而非只换颜色
- 节点像**从主丝线"长出"**，而非排在它旁边
- 可选：鼠标移动时 spine 做 1–2px **反向视差**（"漂浮"最廉价有效的暗示）

### 5.2 hover 整条展开（救可发现性）

> 修订 `design-notes.md` §5「hover 滑出标签」。当前是"hover 谁显谁"，一次只看得清一个。

- 决策：**hover 整条 rail → 全部标题一起滑出**（macOS Dock / VS Code 活动栏展开那种）。
- 收益：即便没记住图标含义，移上去一眼全貌。配合 spine 收到 5 项，展开态非常干净。

## 6. dashboard = 监控画布（widget 化）

> 重定义首页：从"一张好看的有机体图"→ **"监控土壤环境 / 菌丝生长"的可组合画布**。

> **⚠️ SUPERSEDE（2026-06 实现期定稿，#125 / PR #129）**：本节"网格吸附拖拽（方案 B）"
> 经与设计反复对样板后**被推翻**，落地采用 **单一画布 · 模板化漂浮 widget**：
> - **有机体即唯一大地**：铺满视口的背景；删除"正常/全屏"双模式、盒子 hero、`react-grid-layout`。
> - **widget = 漂浮玻璃仪表**：自由摆放（绝对坐标，落点轻吸附半单位网格），不再是网格吸附流。
> - **尺寸 = 离散模板 W×H 单位**：`1×1 / 2×1 / 1×2 / 2×2 / 2×4 / 4×1 …` + 曲线大档 `4×2 / 6×2 / 8×2 / 6×3 / 8×4`；
>   每个 widget 声明支持哪些模板 + 默认档，按内容选最合适（大数字/曲线天生更宽）。
> - **相机**：编辑态可 pan / 滚轮 zoom / ⤢ 适配取景；位置存画布内分数坐标（跨屏按比例 + 夹取，不自动 reflow，靠适配兜底）。
> - 代价认账：这是**空间画布**不是响应式 reflow 仪表盘——小屏靠相机够得着 + 一键适配，换取"无限大地"的直接感。
>
> 实现：`components/dashboard/{dashboard-canvas,canvas-config,widgets,tag-organism}.tsx`；持久化 `{ version, items:[{i,tpl,x,y}], cam }`。
> 下面原"方案 B / bento 等高"段落保留作历史决策轨迹。

- **widget 目录**：预设组件（分组总数 / 标签增长 / 审核积压 / 注册速率 / 有机体图 / 活动流 / 重复检测 / 使用热榜 …），由系统设计，不是用户裸搭。
- **bento 多尺寸**（已决策）：widget 有 `1×1 / 2×1 / 2×2` 等尺寸档（有机体图天生比数字卡大），
  **统一的是"质感"（半透明 / 同款圆角·模糊·留白 / 小巧精美），不是"高度"**。
  —— 字面"等高"会把画布拉回规整网格，与"无限自由"打架，故弃。
- **网格吸附拖拽（方案 B，已决策）**：拖动但自动对齐 / 不重叠 / 可响应式（react-grid-layout 路子）。
  放弃"真·无限自由定位"（会乱、会重叠、几乎无法响应式）。
  **"无限大地"感由背景画布（有机体/关系图铺满）承载，不靠让卡片拖到犄角旮旯。**
- **直接操作**：拖拽排布**在 dashboard 上**做（进"编辑模式"直接拖，Notion/Grafana 式）；
  "可选哪些 widget / 全局默认布局"放设置面。
  —— 绕到设置里去配置布局 = 又变回传统后台的间接感。
